Getting Started

Based on the FEMA lesson template, this application allows creators to build new lessons and steps from a dynamic interface. This page will walk you through the process of setting up your development environment, creating your own pages, and deploying into production.

The file structure of this application varies slightly from the one provided by FEMA. FEMAs samples are incompatible with modern versions of Adobe Dreamweaver, and are not named so finding the one you want can be a complicated task.

To further ease development, a Page Builder is included with this template which allows a user to select the lesson page they want to build, and can interactively generate the necessary steps to integrate their page. If users would rather just copy and paste the sample HTML however, they are certainly welcome to do that.

Development

This template supports many different development environments, from Adobe Dreamweaver, to the text editor of your choice.

Adobe Dreamweaver

To setup Adobe Dreamweaver to be used as your main development IDE, simply open the app/ folder and Dreamweaver should handle the rest for you. HTML files should be editable via code or Live viewer.

  • Supports code editing, as well as live editing
  • Can view pages such as the lesson list which can't be done by simply opening the HTML file in your browser (due to security restrictions)
  • Easily deploy to FTP when complete
  • Cannot use the interactive page builder

Atom/Sublime Text/Notepad and other text editors

Through the use of gulp, you are able to use the editor of your choice when creating your own lessons. To do use, users must install node.js on their machine, and afterwards running npm install in this folder. Be warned that the first time you run this, you may have to wait a few minutes for everything to download.

Once setup, simply run gulp serve in your command prompt and you should see something similar to the image on the right. Once this is complete, go to http://localhost:3000/index.htm and you should be ready for live development.

  • Supports live reloading when saving a file
  • Can be used to extend the Page Builder to add more templates if desired
  • Faster design process
  • Free, no Dreamweaver licenses required
  • Significantly less crashes than Dreamweaver

Usage

The Page Builder will walk you through the process of creating a new lesson and adding steps to it, but here is a short summary of what is required to build a new lesson from scratch:

imsmanifest.xml

The imsmanifest.xml file contains the required lesson information which preserves the order and description of each lesson. To add a new lesson or lesson step, this file must be edited each time.

If you are using the page builder, the required XML markup will be provided to you based on your provided input.


Setting your applicationRoot path

The Page Builder aims to be as set-and-forget as possible, but if you have installed this on a non-root URL, you will have to configure the applicationRoot variable on both the app/page-builder/index.html and app/page-builder/new-step.html files.

For example, if you have installed your instance on http://www.myurl.com/fema, then set your applicationRoot variable to /fema. It is important to set this so that the Page Builder correctly loads the configurator fields.

Deployment

Pushing your customized lesson pages to your production server is a simple process. Just follow the steps below and you should be set.

  1. Connect to your production server (either via FTP, SSH or web interface)
  2. While optional, it is recommended to back up your existing lesson instance before continuing.
  3. Copy and paste the entire contents of the app/ folder
  4. Overwrite any files that need to be overwritten.
  5. Once done uploading, test to make sure everything was uploaded correctly
  6. If it looks good, you are all done.

This can often times be accomplished directly through Dreamweaver if you have your production server details all filled in. Otherwise, using a program such as FileZilla for FTP uploads or PuTTY for SSH uploads is recommended.
