Lesson 1: Introduction

Lesson 1: Introduction

Welcome to the Independent Study Course Template (Version 3.0). This template course will serve two purposes:

  • A reference with guidelines, how-tos, and materials
  • An example/template of the proper way to create an Independent Study (IS) course for the Emergency Management Institute (EMI).

Follow the guidelines on each page to develop your course.

Course Overview

This course serves as a template for EMI's Independent Study courses as well as a guide to developing courses based upon the template (itself). The how-to's, guidelines, and examples in this course assume a basic understanding of HTML.

Objectives: At the end of this course, you will be able to:

  • Develop a course containing the proper architecture and layout.
  • Develop a course that follows 508 requirements and guidelines.
  • Use appropriate template examples to include your own content in a new course.
  • Create a page containing a video in the correct format.
  • Create pages for each of the following types of Knowledge Checks: True/False, Multiple Choice, Multiple Answer, Ordering, Text Areas, Yes/No Radio
  • Utilize appropriate layouts while developing various types of content including: basic text, internal links, external links, lists, images, and interactive content.

Screen Features

  • Select the Close button to exit this window and access the menu listing all lessons of this course. You can select any of the lessons from this menu by simply selecting the lesson title.
  • Select the Glossary button to look up key definitions and acronyms.
  • Select the Help button to review guidance and troubleshooting advice regarding navigating through the course.
  • Track your progress by looking at the Progress bar at the bottom of each screen.
  • Follow the instructions that appear on each screen in order to proceed to the next screen or complete a Knowledge Review or Activity.
  • Select the Back or the Next buttons at the bottom of screens to move backward or forward in the lesson. You may also use the "Navigation" panel located on the left side of the screen to navigate between pages and lessons. Note: If the Next button is dimmed, you must complete an activity before you can proceed in the lesson or use the left hand "Navigation" to bypass it.

Navigating Using Your Keyboard

Below are instructions for navigating through the course using your keyboard.

  • Use the "Tab" key to move forward through each screen's navigation buttons and hyperlinks, or "Shift" + "Tab" to move backwards. A box surrounds the button that is currently selected.
  • Press "Enter" to select a navigation button or hyperlink.
  • Use the arrow keys to select answers for multiple-choice review questions or self-assessment checklists. Then tab to the "Submit" button and press "Enter" to complete a Knowledge Review or Self-Assessment.
  • Warning: Repeatedly pressing "Tab" beyond the number of selections on the screen may cause the keyboard to lock up. Use "Ctrl" + "Tab" to deselect an element or reset to the beginning of a screen's navigation links (most often needed for screens with animations or media).
  • JAWS assistive technology users can press the "Ctrl" key to quiet the screen reader while the course audio plays.

Basic Structure

Most courses for FEMA are set up in the same structure. There is a splash screen that leads the student to a course menu. From there, they select a lesson and move through all of that lesson's screens. While all of this looks transparent to the student, it is set up in a hierarchy of files. Before you create a course, it is best to understand how everything will fit together.

If you took the files from the main directory and turned them into a hierarchical bulleted list, a small two-lesson course might look something like this:

Renaming Files

The files in this sample course all begin with the prefix "SMPL." As you develop your own course, you should change this prefix to something more meaningful to that specific course. For example, if your course is a NIMS course you may have files named "NIMS0101080.htm." Please note, as you use our sample template, you may need to change links from inside the template files as well so that they reference files with the new prefixes.

Main Files (Usage)

The following files must be included in every course. Click on each for more information about it's usage:

ismanifest.xml

The ismanifest.xml file holds the structure and file hierarchy. It contains references to all page-content files. The references to each of these resources are required. This file is called upon by the "SMPLIndex.htm" file and the information within helps build the actual course. It is important that this file is formatted correctly. Open up the "ismanifext.xml" file and add your lessons and all resrouces in the format that is used within this template.

index.htm

The first file that users will see is the splash screen. This file should always be named index.htm. It contains a collage of images related to the current course along with the course code and title. Follow the format found in this example template.

indexMenu.htm

The Course Menu file should be named indexMenu.htm. This is the screen where the student gets a list of all of the lesson titles and links to the lessons. In a single-lesson course, the index.htm page directly launches the lesson. If you have properly developed/modified the ismanifest.xml file, you should not have to make any changes to the indexMenu.htm file. This file automatically pulls the lessons (codes, titles, and links) from the ismanifest.xml file.

SMPLIndex.htm

The final "main structure" file is the SMPLIndex.htm file. This acts as the main page for all content. It is used as a holder/frame for each individual page. These individual pages are loaded within. This can/should be renamed to something appropriate and related to the current course. Again, the contents of this file should not need to be modified if the ismanifest.xml file was developed correctly. It will automatically pull the individual page files from the <resources><resource> section for each lesson.

Installing and Using the Template Files

To begin developing a new course, we recommend creating a new folder for the course and then copying all the sample files there, and then do the following:

  1. Rename all the files (change SMPL01 to the new prefix (i.e. NIMS01)).
  2. Create the new site in Dreamweaver or your HTML editor of choice.
  3. Perform a Search and Replace to search the source code and change all instances of SMPL01 to the new prefix (i.e. NIMS01).
  4. Empty out the /assets folder.
  5. Delete all the non-boilerplate files from the root (SMPL0101070.htm and subsequent).

Be sure to leave all the *.xsd and *.xml files. These are the SCORM files.

File Naming Conventions

The standard page filename is formatted like the following: SMPL0101010.htm

This filename breaks down in the following way (see bold and parenthesis)::

(SMPL)0101010.htm = This can be a two- to four-character string abbreviation of the course title.
SMPL(01)01010.htm = This is a two-digit number corresponding to the course module.
SMPL01(01)010.htm = This is a two-digit number corresponding to the lesson number.
SMPL0101(010).htm = This is a three-digit number corresponding to the page number. Use 010 for page 1, 020 for page 2, 100 for page 10, and so on. This gives you room to grow later; that way if you need to add a page in the middle of a lesson after the lesson has been programmed, you can create a filename with a number between the two existing screens.

Editing the ismanifest.xml File

The ismanifext.xml file must be properly structured and include specific references for the course to function properly. This is required for to create a SCORM compliant course. Follow the example found within this course and use the following information to update your ismanifest.xml file.

This XML file is broken down into various required nodes. Place the appropriate information at each node path shown below:

  • Title: Manifest -> Metadata -> imsmd:lom -> imsmd:general -> imsmd:title ->imsmd:langstring
  • Description: Manifest -> Metadata -> imsmd:lom -> imsmd:general -> imsmd:description ->imsmd:langstring
  • Keywords (multiple allowed): Manifest -> Metadata -> imsmd:lom -> imsmd:general -> imsmd:keywords ->imsmd:langstring
  • Course Title (again): Manifest -> Organizations -> Organization -> Title
  • Lessons (multiple allowed): Manifest -> Organizations -> Organization -> Item
  • Lesson Pages (multiple allowed): Manifest -> Resources -> Resource -> File

Note: For each lesson you will have a resource and multiple files underneath of that resource. Each individual file represents a page/slide within that lesson.

Using the Templates

You may use Dreamweaver to create files from templates; however, this is not required. Simply follow the steps below:

  1. Open the appropriate template (.dwt) in the html editor of your choice.
  2. Look for all editable areas - Editable areas are marked by the following HTML comment tags"<!-- templatebegineditable name="PageHeader" -->" and <!-- templateendeditable -->" where "PageHeader" may differ.
  3. Remove content in all editable areas and replace with new content that should be displayed.
  4. Save as (.htm) file with appropriate name.

508 Compliance Guidelines

All electronic course content must meet EMI's 508 guidelines. The information below is a quick guide that contains common issues and suggestions to ensure that your content is compliant. EMI's full testing process can be found here: 508 - Developer Resources Page.

  • Alt Tags - Alternative text is required and must accurately describe the image. If text is found within the image, that full text must be included in the alt text.
  • Tables
    • Properly identify all headers
    • Do not use images of tables
  • Videos - Closed Captioning and Audio Descriptions
  • Forms - Properly associate labels to form elements
  • Do not modify the template - The template was built and tested for 508 compliance. Only modify "content" areas to ensure that it remains compliant.

Best Practices and Standards

When building content, follow the guidelines below:

  • Avoid using tables for layout purposes. Tables should be used for data purposes only unless necessary.
  • Avoid using inline styles. Use CSS classes supplied to style content instead.
  • Responsive content - Allow your content to change size based upon the browser size. Try not to enforce image height and width.
  • Do not use Adobe Flash
  • Browser support - Your content should display properly in the following web browsers: Internet Explorer 9 and above, Google Chrome, Firefox, Safari
  • Test functionality in all supported browsers.

The National Response Framework (NRF):

  • Is a guide to how the Nation conducts all-hazards response.
  • Builds upon the GDN coordinating structures to align key roles and responsibilities across the Nation, linking all levels of government, nongovernmental organizations, and the private sector.
